# Security Policy

## Supported version

Security fixes are assessed for the current `0.9.x` macOS beta line. Older versions may not receive fixes.

## Reporting a vulnerability

Please do **not** publish vulnerabilities, proof-of-concept exploit code, API keys, or confidential document content in a public issue.

Use the repository's [private vulnerability reporting flow](https://github.com/wangml1e4/md-html-reader/security/advisories) and select **Report a vulnerability**. Include the affected version, impact, concise reproduction information, and any proposed mitigation. This starts a private report visible to the maintainers.

## What happens next

The maintainer will aim to acknowledge a valid report, assess its impact, and coordinate a fix before public disclosure. The project will credit reporters only with their permission.
