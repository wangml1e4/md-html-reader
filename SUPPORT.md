# Support

## Choose the right channel

| Need | Where to go |
| --- | --- |
| Reproducible defect | [Bug report form](https://github.com/wangml1e4/md-html-reader/issues/new?template=bug-report.yml) |
| Product idea or improvement | [Feature request form](https://github.com/wangml1e4/md-html-reader/issues/new?template=feature-request.yml) |
| Experience evaluating the macOS beta | [Beta feedback form](https://github.com/wangml1e4/md-html-reader/issues/new?template=beta-feedback.yml) |
| Installing, building, or contributing | [CONTRIBUTING.md](CONTRIBUTING.md) and the README |
| Security vulnerability | [SECURITY.md](SECURITY.md) |

## Before opening a report

- Search existing issues first.
- State the app version, macOS version, expected result, and actual result.
- For an issue involving Markdown or comments, reduce the report to a non-confidential sample when possible.
- Never attach API keys, private documents, or personal information.

MD+HTML Reader is a macOS beta. The current limitations are documented in [BETA_LIMITATIONS.md](BETA_LIMITATIONS.md).
