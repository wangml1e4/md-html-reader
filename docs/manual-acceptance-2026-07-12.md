# 未保存内容保护验收记录（2026-07-12）

- 应用：`MD+HTML Reader 0.9.0`
- 构建：非 E2E 调试 `.app`
- 工作区：`/tmp/markdown-html-manual-acceptance/workspace`

## 已验证结果

| 验收项 | 操作与证据 | 结果 |
|---|---|---|
| 原生未保存提示 | 修改 Markdown 后切换文件，出现 Tauri 原生 `未保存的更改` 警告，提供 `No` / `Yes` 两个按钮 | 通过 |
| 提示期间暂停自动保存 | 保持警告框打开超过 3 秒，磁盘文件未出现 `Paused autosave discard token.` | 通过 |
| 确认放弃 | 点击 `Yes` 后切换到目标文件；源文件仍包含基线文本，未写入临时编辑标记 | 通过 |
| 取消关闭生命周期 | 单元测试模拟 `onCloseRequested`，编辑器拒绝放弃时调用 `preventDefault`，允许时不阻止，并在卸载时注销监听器 | 通过（自动化） |

## 工具边界

Computer Use 可以读取并操作 Tauri 原生警告框，但当前无法通过辅助功能接口真正激活 macOS 的 `Close Window` 菜单项或窗口关闭按钮。因此，本轮没有把“真实点击关闭按钮后出现提示”标为人工通过；该连接由应用生命周期测试覆盖，原生对话框及放弃行为由上述真实应用操作覆盖。

## 结论

生产构建中的原生二选一对话框、自动保存暂停和放弃后磁盘不变均已闭环。窗口关闭事件的接线由自动化测试验证，保留上述 Computer Use 工具限制说明。
